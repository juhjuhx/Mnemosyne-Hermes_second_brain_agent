## 4. Research outcomes

For each RQ, the answer + source + confidence:

### RQ1: Hermes Agent
- **Answer**: Yes. `NousResearch/hermes-agent` is at 162K★ (mid-2026), MIT-licensed, MCP-native, agentskills.io compatible.
- **Source**: GitHub, Nous Research blog
- **Confidence**: High
- **Implication**: Drop v3's self-built FastAPI agent. Use Hermes + 3 example skills.

### RQ2: Mnemosyne
- **Answer**: Yes. `AxDSan/mnemosyne` is a maintained SQLite-only memory layer.
  - LongMem benchmark: 98.9% (claims)
  - BEAM benchmark: 65.2% (claims)
- **Source**: GitHub, AxDSan README
- **Confidence**: Medium-high (need to verify with real benchmark in Phase 4)
- **Implication**: Drop v3's self-built memory layer. Use Mnemosyne + SQLite.

### RQ3: Small LLM for M1
- **Answer**: `unsloth/gemma-4-e2b-it-GGUF` Q4_K_XL ~2.5GB.
  - Gemma 4 was released in 2026, E2B = efficient 2B params
  - GGUF + Metal via Ollama works on M1
  - Fits in 8GB with macOS+Hermes+indexer headroom
- **Source**: Hugging Face, Ollama registry
- **Confidence**: High
- **Implication**: Use Gemma 4 E2B on M1 for "fast/cheap" tasks; route heavy tasks to workstation.

### RQ4: Medium LLM for A770 16GB
- **Answer**: `Ex0bit/Qwen3.6-27B-PRISM-PRO-DQ` Q4 13.7GB.
  - Qwen 3.6 released 2026-04-16
  - PRISM = self-speculative decoding via bundled MTP head
  - 80 tok/s → 121 tok/s with MTP
  - 13.7GB fits A770 16GB with 2.3GB to spare for KV cache
- **Backup**: `mudler/Carnice-Qwen3.6-MoE-35B-A3B-APEX-MTP-GGUF` I-Compact 17GB (slightly over budget, swap if 27B underperforms)
- **Source**: Hugging Face, Qwen GitHub
- **Confidence**: High for 27B PRISM; Medium for 35B-A3B
- **Implication**: Use 27B PRISM as default on workstation. Keep 35B-A3B for benchmark comparison.

### RQ5: Open WebUI
- **Answer**: Yes. Open WebUI supports any OpenAI-compatible API endpoint via "Connections" → "OpenAI API".
  - Hermes Agent ships an OpenAI-compatible server on `127.0.0.1:8642`
  - 5-minute setup with model name "Hermes Agent"
- **Source**: Open WebUI docs, Nous Research docs
- **Confidence**: High
- **Implication**: Use Open WebUI as primary chat UI. Add Hermes CLI for terminal access.

### RQ6: PySceneDetect
- **Answer**: Yes, very. v0.7 released 2026-05-03.
  - AdaptiveDetector F1=91.6%
  - ThresholdDetector F1=88.2%
  - ContentDetector F1=85.4%
- **Source**: PyPI, GitHub releases
- **Confidence**: High
- **Implication**: Keep v3's video slicing pipeline. Upgrade to v0.7 + AdaptiveDetector.

### RQ7: llama.cpp + Vulkan + PRISM-DQ
- **Answer**: Yes.
  - llama.cpp Vulkan backend stable since b4102
  - PRISM-DQ has bundled MTP head in single .gguf
  - `llama-speculative` accepts self-speculative config
- **Source**: llama.cpp repo, Ex0bit README
- **Confidence**: High
- **Implication**: Use llama-server with `--model Qwen...PRISM-DQ.gguf --speculative-model-draft MTP-head` config.

### RQ8: Tailscale
- **Answer**: Yes, still the best. Headscale (OSS alternative) is also viable.
- **Source**: Tailscale 2026 status page
- **Confidence**: High
- **Implication**: Use Tailscale. 127.0.0.1 binding on all services. Tailscale ACLs for the mesh.

## 5. Decision synthesis

After research, the stack was determined. The reasoning chain:

```
M1 has 8GB RAM, 256GB SSD, Metal GPU
  └─→ Can run a 2.5GB Q4_K_XL GGUF via Ollama
       └─→ Gemma 4 E2B is the right size
            └─→ But 2B params is too small for deep reasoning
                 └─→ Need to route complex queries elsewhere
                      └─→ Workstation has A770 16GB, 48GB RAM
                           └─→ 27B Q4 = 13.7GB fits, with 2-3GB KV
                                └─→ Qwen 3.6 PRISM-DQ with MTP = best choice
                                     └─→ Network: Tailscale mesh
                                          └─→ Agent: Hermes (MCP-native)
                                               └─→ Memory: Mnemosyne (SQLite)
                                                    └─→ Frontend: Open WebUI
                                                         └─→ Skills: agentskills.io spec
```

## 6. What we **did not** research (and why)

- **Fine-tuning Qwen 3.6 on user data** — out of scope; v4 plan uses base models.
- **Multi-GPU on workstation** — A770 16GB is enough for 27B Q4; no need for tensor-parallel.
- **Federated / multi-user setups** — explicit non-goal; v3 invariant is single-user.
- **Cloud fallback** — explicit non-goal; v3 invariant is local-first.
- **Face / voice emotion monitoring** — explicit non-goal; v3 invariant forbids it.
