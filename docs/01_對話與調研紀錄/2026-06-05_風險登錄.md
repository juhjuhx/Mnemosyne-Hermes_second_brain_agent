## Detailed risk descriptions

### R01 — M1 RAM exceeds 7.5GB

**Trigger**: Indexer + Hermes + Ollama + macOS all running simultaneously.

**Impact**: Kernel kills Ollama → Hermes falls back to no-model mode.

**Mitigation**:
- Phase 1: pre-flight `vm_stat` + `ps` check before starting Indexer
- Phase 3: Indexer pauses when Ollama is mid-request
- Phase 4: monitor via Prometheus + alert

### R02 — Workstation VRAM exhaustion

**Trigger**: Concurrent requests exceed KV cache budget.

**Impact**: llama-server returns 503 / OOM-killed.

**Mitigation**:
- Phase 1: `--parallel 4` (tested at 2; cap at 4)
- Phase 4: load test at 4×, 8×, 16× concurrent
- Phase 5 fallback: drop to 14B Q4

### R13 — Eval set has ground-truth errors

**Trigger**: 100 queries × ~3 annotations each = 300 cells, ~5% error rate = 15 wrong labels.

**Impact**: "Eval is good" is false positive; system regresses unnoticed.

**Mitigation**:
- Phase 4: 2-pass annotation (annotator A, then B re-checks A)
- Inter-annotator agreement ≥ 0.85 required
- Spot-check 10% of queries monthly
