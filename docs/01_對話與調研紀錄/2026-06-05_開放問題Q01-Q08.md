## Q01 — Does M1 run Hermes, or just connect?

**Context**: Hermes Agent can run on either machine. M1 has 8GB RAM, workstation has 48GB.

**Options**:
- **A**: M1 runs Hermes (MCP coordinator + local Ollama as one of its models)
- **B**: Workstation runs Hermes (MCP coordinator + M1 Ollama as remote model)

**Tradeoffs**:

| | A: M1 runs Hermes | B: Workstation runs Hermes |
|---|---|---|
| Single point of failure | M1 (we'd lose access) | Workstation (worse — heavier) |
| Tailscale dependency | M1 always on | M1 must wake on demand |
| Latency (M1 local task) | ~0ms (localhost) | ~5-50ms (LAN) |
| RAM pressure on runner | +0.5GB on M1 | +0.5GB on workstation (no concern) |

**Default for v4**: A (M1 runs Hermes) — failsafe if workstation is off.

**Resolve in**: Phase 3 (Hermes install). Re-evaluate if M1 RAM pressure mounts.

## Q02 — Snippet vs full-text indexing

**Context**: Should Hermes's `mcp-filesystem` return full file contents, or snippets with links?

**Options**:
- **A**: Always snippets (200-500 chars) + path. User clicks to open.
- **B**: Full text when file is small (<10MB), snippet otherwise.
- **C**: User-configurable per query.

**Default for v4**: B (auto).

**Resolve in**: Phase 3 skill implementation. Likely A wins after user testing.

## Q03 — Tailscale vs Headscale

**Context**: Tailscale's hosted control is convenient but technically a cloud dependency.

**Options**:
- **A**: Tailscale hosted (current default)
- **B**: Headscale (OSS control server, self-hosted on workstation)

**Default for v4**: A (faster setup; "cloud dependency" is only control plane metadata, not data).

**Resolve in**: Phase 0 if user has strong "no cloud at all" preference.
