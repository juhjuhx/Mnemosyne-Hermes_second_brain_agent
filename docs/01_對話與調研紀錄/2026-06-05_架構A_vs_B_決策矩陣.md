## Decision matrix

| Dimension | A (M1 alone) | B (M1 + workstation) | Winner |
|---|---|---|---|
| **Heavy LLM quality** | None / cloud-dependent | 27B Q4 with MTP speculative decoding | **B** |
| **Privacy** | Mixed (cloud for heavy) | 100% local | **B** |
| **Hardware cost** | $0 (existing) | $0 (existing) | Tie |
| **M1 RAM headroom** | ~1.15 GB | ~1.15 GB | Tie |
| **Setup complexity** | Low | Medium (Tailscale + 2 services) | **A** |
| **Network dependency** | None (offline OK) | LAN (Tailscale; works offline if same subnet) | **A** slightly |
| **Inference speed (M1, easy)** | ~25 tok/s | ~25 tok/s (same Ollama on M1) | Tie |
| **Inference speed (heavy, workstation)** | N/A | ~80-121 tok/s (PRISM-DQ) | **B** |
| **Failure modes** | Single point | Two points, but Tailscale mesh | Slight **A** |
| **Power draw (M1)** | 15-20W | 15-20W | Tie |
| **Power draw (workstation)** | 0W | 80-150W | **A** |
| **Heat / noise** | Fanless | Workstation fan noise | **A** |
| **Scalability** | Bounded by 8GB | Can swap 27B for 35B-A3B or smaller | **B** |
| **Cloud lock-in** | High (need cloud for heavy) | None | **B** |
| **"Second brain" depth** | Shallow (2B) | Deep (27B) | **B** |
| **v3 invariant compatibility** | Compatible | Compatible | Tie |
| **Maintenance burden** | 1 machine | 2 machines, 1 mesh | Slight **A** |
| **Edge perception quality** | Same on both | Same on both | Tie |
| **Future RPC split (Phase 5)** | Not applicable | Natively supported | **B** |
| **Open-source ecosystem fit** | Smaller | Better (Hermes/Qdrant/Tailscale) | **B** |

### Tally

- **B wins**: 9 dimensions
- **A wins**: 5 dimensions (mostly operational concerns)
- **Tie**: 4 dimensions
- **B's wins are strategic** (privacy, quality, future-proofing)
- **A's wins are tactical** (simplicity, power, noise)

---

## Risk-adjusted decision

Even weighting wins by 1-2-3:

| Dimension weight | A score | B score |
|---|---:|---:|
| Privacy (×3) | 1 | 3 |
| Heavy LLM quality (×3) | 0 | 3 |
| Future scalability (×2) | 1 | 2 |
| v3 invariant compatibility (×3) | 3 | 3 |
| Setup simplicity (×1) | 1 | 0 |
| Power/noise (×1) | 1 | 0 |
| Inference speed (heavy) (×2) | 0 | 2 |
| Cloud lock-in (×2) | 0 | 2 |
| **Total** | **9** | **17** |

**B wins 17 vs 9.**

---

## Conditions under which A would win

A is the right choice if **any** of these are true:

- Workstation is not available / sold / broken
- M1 is upgraded to 16GB+ (Mac mini M2 Pro, Mac Studio M2 Max, etc.)
- User does not need heavy reasoning
- Power draw is a hard constraint (e.g. running on battery in a van)

We can revisit A later (e.g. by changing `M1 main LLM` to a larger Qwen) and keeping
`workstation` as an optional accelerator.
