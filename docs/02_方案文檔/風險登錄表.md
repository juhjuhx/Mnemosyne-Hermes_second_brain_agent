## How to read this table

| Column | Meaning |
|---|---|
| **#** | Risk ID (R01-R20) |
| **Risk** | One-line description |
| **P** | Probability: L/M/H |
| **I** | Impact: L/M/H |
| **Phase** | Which phase the risk is most likely to surface |
| **Owner** | Who is responsible for monitoring + mitigation |
| **Status** | Open / Mitigated / Accepted / Closed |
| **Mitigation** | Concrete action(s) |

## Risk lifecycle

When a risk materializes (or is found to be a non-issue):

1. Update `Status` to `Mitigated` (if mitigation worked) or `Closed` (if no longer relevant).
2. Add a row to `CHANGELOG.md` under `[Unreleased]` with the change.
3. If mitigation was costly, capture a "lesson learned" in `docs/07_更新日誌/`.
